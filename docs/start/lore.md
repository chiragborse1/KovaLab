# The Lore of Kova 🦄

_A short origin story for the current Kova identity._

## The short version

Kova is the current name, voice, and visual identity for the terminal agent and Gateway. It grew out of earlier project names during a fast-moving rebrand, and the goal of that rebrand was simple: one product name that works cleanly across the CLI, docs, channels, plugins, and onboarding.

## What Kova means

Kova is the operator-facing product identity for the whole system:

- the gateway you run
- the terminal chat and CLI commands you use
- the agent you message through your channels
- the docs and onboarding flow that explain it

The tone is direct, practical, and a little playful, but never vague.

## The rebrand

The project went through earlier naming phases before landing on Kova. Those older names still exist in parts of the codebase for compatibility, package history, and migration safety, but the user-facing product is Kova.

That distinction matters:

- **User-facing surfaces** should say **Kova**
- **Compatibility contracts** may still use older names until a safe migration path exists

## Product voice

Kova should sound like:

- clear
- capable
- concise
- technically sharp

Kova should not sound like:

- brand-confused
- mascot-heavy
- lore-first when the user needs operational guidance

## Symbols and visual language

The current visible mascot direction is the unicorn:

- `🦄` for playful user-facing shorthand
- the Kova logo for UI, docs, icons, and onboarding

Avoid falling back to older mascot language or visuals on user-facing surfaces.

## Why this page exists

This page is here so docs, UI copy, and onboarding text stay aligned. If a sentence reads like it belongs to an older identity, it probably does.
